# ATIVIDADE 6 — O shell como JCL moderno

# O slide da genealogia mostrou que o JCL usava comandos como
# // ASSIGN SYS004,TAPE para associar um job a um dispositivo.
# No shell moderno, qual mecanismo cumpre papel equivalente?

Resposta:
No shell moderno, esse papel é feito pelos redirecionamentos
e pelos pipes do terminal.
Eles conectam arquivos, comandos e programas usando símbolos
como >, < e |, igual aconteceu no script relatorio.sh.

# ATIVIDADE 7 — Make

# 1. O monitor histórico verificava a conclusão de cada job antes
#    de despachar o próximo. No make, qual mecanismo cumpre esse
#    papel? Como o make sabe que não precisa recompilar um arquivo
#    que não mudou?

Resposta 1:
O make usa as dependências e os timestamps dos arquivos.
Ele verifica a data da última modificação.
Se o executável já estiver mais novo que o arquivo fonte,
ele entende que nada mudou e não recompila.

# 2. O monitor unificava "supervisor, carregador e rotinas de I/O
#    sob uma interface textual". O que o Makefile unifica no fluxo
#    de desenvolvimento moderno?

Resposta 2:
O Makefile unifica várias tarefas do desenvolvimento em um só lugar,
como compilar programas, executar scripts, gerar relatórios
e limpar arquivos do projeto.
Assim o processo fica mais organizado e automático.
